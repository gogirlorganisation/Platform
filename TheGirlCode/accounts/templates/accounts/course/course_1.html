Operators

The basic symbols used by Python to perform mathematical operations are called operators:

  • Addition (+)
  • Subtraction (-)
  • Multiplication (*)
  • Division for Quotient (/)
  • Modulo (%)

What does Modulo do, you may ask? Well it’s quite simple.
Try figuring out what it does:

1 % 4 = 1
15 % 4 = 3
10 % 4 = 2

Got it? Yes, you’re right! Modulo returns the remainder.

4 x 0 + 1
4 x 3 + 3
4 x 2 + 2

Good work!

Python follows BODMAS rules. For example, if you enter:

5+6*10

5 and 10 are multiplied and 6 is added to the result - the answer is 56 and not 110.

We use brackets () to correct this. The bracket tells Python to perform the operation inside it first.

(5+6)*10

Here, the operation inside the brackets is performed first, so 11 is multiplied to 10. Thus, it equals 110.

To raise a number to a power we can simply use **

2 ** 2 = 4
3 ** 2 = 9

There are other type of operators as well, these are called boolean operators. These handle the expression, True and False.

Just like the addition (+), subtraction (-), multiplication (*), and division (/) operators for math problems, boolean problems have their own operators:

  • > : greater than
  • < : less than
  • >= : greater than or equal to
  • <= : less than or equal to
  • == : equivalent
  • != : not equivalent

Why two == and not =?
Well = holds another meaning that we will see later.

Booleans expressions are either True or False.

9 > 10

Output: False

3 + 3 < 10

Output: True

We can even combine these expressions using and, or and not.

AND
If both expressions are True it returns True, else it returns False.

3 + 4 == 7 and 8 < 9

3 + 4 is 7 so the first expression is True.
8 is less than 9 so the second expression is also True.

Hence, the output is True .

3 + 4 == 7 and 10 < 9

3 + 4 is 7 so the first expression is True.
10 is not less than 9 so the second expression is False.

Hence, the output is False .

OR
If both expressions any one expression is True

3 + 4 == 7 or 8 != 9

3 + 4 is 7 so the first expression is True.
8 doesn't equal 9 so the second expression is also True.

Still, unlike and the output is True .

4 + 4 == 7 or 10 < 9

4 + 4 is 8 so the first expression is False.
10 is not less than 9 so the second expression is False.

Hence, the output is False .

NOT
Not just returns the opposite of the given result. If the expression is True it will reutrn False and vica versa.

not (3 + 4 == 7)

3 + 4 is 7 so the first expression is True but because of the not it gives False.

Hence, the output is False .

not (10 < 9)

10 is not less than 9 so the second expression is False but because of the not it gives True.

Hence, the output is True .
